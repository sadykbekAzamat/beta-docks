# Akcent Analytics — Что это и для чего

## О платформе

**Akcent Analytics** — это внутренняя система управления для учебных центров. Она объединяет в одном месте всё, что нужно руководству и сотрудникам: продажи, зарплаты, финансы, расходы и маркетинг. Вместо разрозненных таблиц Excel и переписки в мессенджерах — одна понятная система, где каждый видит только то, что ему нужно.

Система обслуживает несколько учебных центров одновременно и разграничивает данные между ними. На текущий момент это **Akcent Academy** и **Turkey Study**.

---

## Кто пользуется системой

В системе есть несколько типов пользователей, у каждого свои права и возможности:

| Роль | Кто это | Что видит |
|------|---------|-----------|
| **Администратор** | Владелец или руководитель высшего уровня | Всё без ограничений |
| **Финансист** | Бухгалтер, финансовый менеджер | Финансы, зарплаты, FOТ, P&L, маркетинг |
| **Оператор Akcent** | Аналог финансиста для Akcent Academy | Те же возможности, что у финансиста |
| **РОП** | Руководитель отдела продаж | Продажи и зарплаты своих ОПов |
| **ДОП** | Директор отдела продаж | Продажи и зарплаты своих РОПов |
| **Admin Sales** | Менеджер, руководящий Closerами | Продажи и зарплаты своих Closerов |
| **ОП** | Оператор продаж | Только свои чеки и свои финансовые строки |
| **Closer** | Менеджер по закрытию сделок | Только свои чеки |
| **Маркетолог** | Специалист по рекламе | Только маркетинговые отчёты |
| **Support / Sales** | Прочие сотрудники | Минимальный доступ |

Каждый сотрудник при входе видит своё имя и роль прямо на главной странице.

## Что умеет система

### 📈 Продажи и зарплаты

Это главный раздел для руководства отдела продаж — РОПов, ДОПов, Admin Sales, а также для финансистов и администратора.

**Таблица продаж**

Основной экран показывает всех менеджеров со следующими данными за выбранный период:
- имя и роль сотрудника
- принадлежность к компании (Akcent Academy или Turkey Study)
- общая сумма продаж

Данные фильтруются по: месяцу и году, компании, имени сотрудника.

**Виды сделок**

Система разбирает продажи по типам. Каждый тип влияет на расчёт зарплаты сотрудника:

| Тип | Описание |
|-----|----------|
| Пробный урок (trial) | Разовый ознакомительный урок |
| Пробный урок duo (trial_duo) | Пробный урок для пары студентов |
| Пробный offline (trial_offline) | Пробный урок в офлайн-формате |
| Пробный на платформе (trial_platform) | Пробный урок через онлайн-платформу |
| Прямая продажа (direct) | Продажа курса без предоплаты |
| Предоплата (prepayment) | Частичная оплата, бронирование |

**Детали зарплаты каждого сотрудника**

По кнопке рядом с именем открывается подробная карточка зарплаты, в которой видно:
- общее число продаж и расчётная зарплата
- количество проведённых пробных уроков и бонус за них
- **Выплачено** — сколько фактически перечислено на руки
- **Аванс** — выплата за первую половину месяца
- **Бонусы** — дополнительные поощрения
- **Штрафы за первую половину месяца**
- **Штрафы за вторую половину месяца**
- **Пенсионные отчисления**
- Список всех чеков этого менеджера с цветовыми метками статуса (ожидает / подтверждён / отклонён)

Финансист может вручную вносить и редактировать суммы выплаты, аванса, бонусов, штрафов и пенсионных отчислений прямо в этой карточке.

**Вид «Головной / Сводный»**

Для РОПов, ДОПов и Admin Sales есть специальный сводный экран, который одновременно показывает три секции:
- таблица РОПов с их суммарными показателями
- таблица ДОПов
- таблица Admin Sales

Это позволяет руководству одним взглядом оценить ситуацию по всей структуре.

**Экспорт в Excel**

Все данные по продажам и зарплатам за выбранный период можно экспортировать в Excel-файл одним нажатием кнопки. Файл скачивается с уже заполненными данными — готов для архива или ручной проверки.

### 🧾 Мои чеки (для ОПов и Closerов)

Каждый оператор продаж и Closer видит свой личный список чеков — только свои, ничего лишнего.

**Добавление нового чека**

Сотрудник нажимает кнопку «+ Добавить чек» и заполняет:
- последние **5 цифр** номера операции из Kaspi
- тип сделки (пробный, предоплата, прямая продажа и другие)
- компания (Akcent Academy или Turkey Study)
- телефон клиента (опционально)
- месяц, к которому относится продажа

**Статусы чеков**

После добавления чек отображается со статусом. Цвет значка меняется в зависимости от ситуации:
- **Ожидает** — чек добавлен, финансист ещё не проверил
- **Подтверждён** — финансист нашёл соответствующую строку в банке и подтвердил
- **Отклонён** — чек не прошёл проверку (неверные данные, не найден в банке)

Отклонённый чек не учитывается при расчёте зарплаты. Менеджер может видеть причину отклонения или переспросить финансиста.

### 📂 Загрузка Kaspi-выписки

Этот инструмент используется финансистами для загрузки банковских данных в систему.

**Как работает загрузка**

- Интерфейс с зоной перетаскивания файлов: можно перетащить xlsx-файлы прямо мышью или нажать и выбрать через проводник
- Поддерживается одновременная загрузка **до 20 файлов**
- После выбора система показывает список файлов с их размером

**Обработка данных**

После нажатия кнопки «Загрузить и разобрать» система автоматически:
- читает все операции из каждого файла
- определяет категорию каждой операции:
  - **Индивид** — индивидуальное занятие
  - **Групповой** — групповое занятие
  - **Пробный** — пробный урок
  - **Предоплата** — бронирование / частичная оплата
  - **Возврат** — возврат денег клиенту
  - **Другое** — прочие операции
- разделяет операции на **Продажи** и **Возвраты**

**Итоговая статистика**

После загрузки система показывает:
- количество обработанных файлов
- число строк продаж
- число возвратов
- общая сумма продаж

Таблица результатов содержит: дату, категорию, сумму, комиссию Kaspi, способ оплаты и номер операции. Данные автоматически становятся доступны для сопоставления с чеками менеджеров.

### 💰 Финансовые отчёты

Раздел состоит из нескольких вкладок. Видимость вкладок зависит от роли: ОП видит только свои отчёты и вкладку добавления, привилегированные роли видят все инструменты.

**Вкладка «Отчёты» и «Добавить»**

Финансист вводит строки вручную — каждая строка соответствует одной оплате из банка:
- дата операции
- категория (Индивид, Групповой, Пробный, Предоплата, Возврат)
- сумма и комиссия
- номер операции (последние 4 цифры используются для поиска)
- компания

После сохранения система автоматически рассчитывает, какая доля суммы идёт каждому участнику сделки: ОПу, Closerу, РОПу, ДОПу и учителю — в соответствии с настроенными правилами.

**Вкладка «Верификация чеков»**

Это рабочий стол финансиста для проверки чеков, которые добавили менеджеры.

Вверху — **статистическая панель**: три карточки показывают общее количество чеков по статусам — сколько ожидает, сколько подтверждено, сколько отклонено.

**Поиск по последним 4 цифрам**: финансист вводит последние 4 цифры номера операции — и система мгновенно находит соответствующую строку в банковских данных. Это главный инструмент сопоставления чека менеджера с реальным платежом.

Для каждого чека доступны действия:
- **Подтвердить** — чек соответствует банковской строке
- **✓ Akcent** — специальная кнопка для подтверждения оплат, связанных с Akcent Academy
- **Отклонить** — чек не прошёл проверку
- **Вернуть в ожидание** — откат чека в начальный статус из любого другого

Финансист может **редактировать чек** прямо в модальном окне: изменить номер операции, тип сделки, компанию, телефон клиента или месяц.

**Вкладка «Маппинг»**

Наглядное представление связей между чеками менеджеров и строками банковской выписки.

Показываются последние 1000 чеков. Для каждого чека отображается:
- первые 5 символов номера операции, статус, тип, компания, месяц
- **зелёный индикатор «✓ N строк финанс.»** — если чек привязан к банковской строке (и сколько строк)
- **красный индикатор «✗ нет привязки»** — если чек ещё не сопоставлен

Нажав на чек, можно увидеть детали всех привязанных банковских строк: идентификатор, сумма, категория.

### 👥 ФОТ (Фонд оплаты труда)

Раздел для финансистов — учёт фиксированных окладов всех сотрудников компании.

**Сводная таблица**

Вверху страницы всегда видны два итога:
- **общая сумма ФОТ** — суммарный фонд оплаты труда по всем показанным сотрудникам
- **количество сотрудников** в текущей выборке

**Данные по каждому сотруднику**

В таблице для каждого сотрудника хранится:
- полное имя и должность
- компания (Akcent Academy или Turkey Study)
- тип занятости: **Полная**, **Частичная**, **Контракт**, **Стажёр**, **Другое**
- оклад
- день месяца, в который производится выплата
- заметка (произвольный текст, например «выплата в начале месяца»)
- процент от продаж (для сотрудников с переменной частью дохода)

**Управление**

- Добавление нового сотрудника через форму
- **Редактирование прямо в строке таблицы** (inline edit): нажал кнопку рядом со строкой — поля стали редактируемыми, сохранил — всё обновилось
- Удаление записи

Фильтрация по компании позволяет отдельно смотреть ФОТ Akcent Academy или Turkey Study.

### 📊 P&L — Расходы

Учёт всех операционных расходов компании.

**Добавление расхода**

Каждый расход вносится с полями:
- дата
- название (например: «Аренда офиса», «Реклама в Instagram», «Зарплата уборщицы»)
- сумма
- компания (Akcent Academy или Turkey Study)
- заметка (опционально)

**Просмотр и фильтрация**

- Фильтр по **периоду**: указывается дата «с» и «по» — система показывает только расходы в этом диапазоне
- Фильтр по **компании**
- Над таблицей всегда виден **итог** — суммарная сумма расходов за выбранный период

**Редактирование**

Любую строку расходов можно отредактировать прямо в таблице, не открывая отдельное окно: нажать карандаш, изменить нужные поля, сохранить.

### 📣 Маркетинг

Раздел для отслеживания эффективности рекламных кампаний.

**Отчёты по каналам**

Маркетолог создаёт отдельный отчёт для каждого рекламного канала (например: «Яндекс Директ», «Instagram», «ВКонтакте»). Каждый отчёт привязан к конкретному **месяцу и году**.

**Ежедневные данные**

Внутри каждого отчёта — таблица по дням. Для каждого числа месяца вносится:
- количество лидов (заявок), пришедших в этот день
- расход на рекламу в тенге за этот день

**Автоматический расчёт CPL**

Система сама считает **CPL** (Cost Per Lead — стоимость одного лида) для каждого дня и для всего отчёта в целом. Это ключевая метрика эффективности: чем ниже CPL, тем дешевле обходится привлечение одного потенциального клиента.

В сводке по каждому отчёту видно:
- общее число лидов за месяц
- суммарный расход на рекламу
- средний CPL за месяц

**Детализация**

Отчёт можно раскрыть и посмотреть данные по каждому дню в отдельности: число лидов, расход, CPL за конкретный день.

Все отчёты можно редактировать: поменять название канала, добавить или убрать дни, исправить цифры.

### 👤 Сотрудники

Управление всеми пользователями системы. Доступно администратору и финансисту (с ограничениями).

**Два режима просмотра**

Страница работает в двух режимах, переключаемых вкладками:

- **Список** — полная таблица всех сотрудников с их ролью, компанией, контактами и статусом
- **Иерархия** — наглядная структура подчинения: кто чей РОП, кто чей ДОП, кто управляет Closerами

**Таблица сотрудников**

В таблице по каждому сотруднику видно:
- имя и email
- роль в системе
- компания
- к какому РОПу, ДОПу и Admin Sales привязан
- процент от продаж (если применимо)

**Создание зарплат за месяц**

Кнопка **«Создать зарплаты за месяц»** автоматически формирует зарплатные записи для всех активных сотрудников за текущий месяц. Это избавляет от необходимости вручную создавать запись каждому — одна кнопка, и все записи готовы.

**Добавление нового сотрудника**

Форма создания включает:
- email и временный пароль
- полное имя
- роль
- компания
- процент от продаж
- привязка к РОПу, ДОПу или Admin Sales (по их идентификатору)

**Права зависят от роли**: администратор может создавать сотрудников любой роли. Финансист ограничен — он может добавлять только ОПов и Closerов.

**Редактирование**

В режиме редактирования можно поменять имя, роль, компанию, процент и все привязки к руководителям.

**Сброс пароля**

Только администратор может сбросить пароль любому сотруднику. После сброса новый пароль показывается прямо на экране — его нужно скопировать и передать сотруднику.

**Мягкое удаление**

При удалении сотрудника его аккаунт деактивируется, а не стирается навсегда. Вся история продаж и финансовых данных сохраняется в системе.

### 🏢 Компании

Только для администраторов.

- Список всех учебных центров, подключённых к системе
- Можно добавить новую компанию (название, описание) или отредактировать данные существующей
- Все данные в системе — продажи, зарплаты, расходы — привязаны к конкретной компании

### ⚙️ Настройки зарплат

Раздел для настройки правил начисления зарплаты. Доступен администратору.

**Для кого настраивается**

Настройки задаются отдельно для каждой роли в каждой компании:
- ОП
- Closer
- РОП
- ДОП
- Admin Sales
- Оператор Akcent

Это значит, что у ОПа в Akcent Academy и ОПа в Turkey Study могут быть совершенно разные условия.

**Ступени (тиры) зарплаты**

Система поддерживает прогрессивную шкалу: чем больше продал — тем выше процент с каждой сделки.

Пример настройки:
- от 0 до 10 сделок — 5% от суммы
- от 11 до 20 сделок — 7% от суммы
- от 21 сделки и выше — 9% от суммы

Можно настроить любое количество ступеней. Верхняя ступень без ограничения обозначается символом «∞».

**Бонус за пробный урок**

Отдельно задаётся фиксированная сумма в тенге, которую сотрудник получает за каждый проведённый пробный урок, независимо от его стоимости. Например: 2 000 ₸ за каждый пробный.

**Управление настройками**

- Добавление новой настройки для роли и компании
- Редактирование существующей: изменить бонус за пробный, добавить или удалить ступени, изменить проценты
- Удаление настройки
- Фильтрация по компании и роли для удобного поиска

- **Смена пароля**: нужно ввести текущий пароль, затем новый и подтверждение нового

## Главная страница (Дашборд)

При входе в систему сотрудник видит персональное приветствие со своим именем и ролью.

Для руководителей и финансистов автоматически подгружается **сводка за последние 30 дней**:

**Три ключевых метрики:**
- **Общий объём продаж** — суммарная выручка по всем продавцам
- **Сумма Kaspi-отчётов** — сколько прошло через банк
- **Комиссия Kaspi** — сколько из этой суммы удержал банк в качестве комиссии

**Продажи по позициям** — таблица, в которой виден объём продаж и число сотрудников в каждой роли (ОП, Closer, РОП и т.д.).

**Финансовые отчёты по компаниям** — таблица с суммой, комиссией и числом строк отдельно по Akcent Academy и Turkey Study.

Для сотрудников без доступа к общей аналитике (ОП, Closer и другие) дашборд показывает краткое приветствие и предлагает перейти в нужный раздел через меню.

## Безопасность и приватность

- Каждый пользователь входит по персональному логину и паролю
- После нажатия кнопки «Выйти» сессия немедленно закрывается
- Каждый видит **только свои данные** — ОП не может увидеть чеки другого ОПа
- Все пароли хранятся в **зашифрованном виде** — никто, включая администраторов, не видит пароли в открытом виде
- Сброс пароля возможен только администратором вручную
- Доступ ко всем разделам ограничен в зависимости от роли — случайно попасть в чужой раздел невозможно
